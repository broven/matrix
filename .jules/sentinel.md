## 2025-05-15 - [Localhost CSRF / DNS Rebinding Protection]
**Vulnerability:** Malicious websites can use a user's browser to make requests to services running on `localhost` or `127.0.0.1`. If the local service relies solely on the source IP for authentication (a common pattern for "local-only" tools), a website can exfiltrate sensitive data like auth tokens or control the agent.
**Learning:** Standard CORS checks don't prevent simple GET/POST requests that don't trigger a preflight. Even with CORS properly configured, some browser-based attacks can still hit the endpoint.
**Prevention:** Require a custom, non-standard header (e.g., `X-Matrix-Internal: true`) for all sensitive loopback-only endpoints. Browsers *always* trigger a CORS preflight for custom headers. By excluding this header from the CORS `allowHeaders` whitelist, we ensure that a browser will never be able to successfully send a request with this header from a cross-origin context, even if it's hitting a loopback address.
