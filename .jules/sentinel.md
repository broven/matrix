## 2025-04-06 - Localhost CSRF and DNS Rebinding Protection
**Vulnerability:** Sensitive loopback-only endpoints like `/api/auth-info` (which returns the server authentication token) and `/api/local-ip` were accessible to any browser-based request originating from the same machine. Even with a restrictive IP check, a malicious website could exploit the permissive CORS policy or DNS rebinding to steal the authentication token.
**Learning:** IP-based restrictions for loopback are insufficient for preventing browser-based cross-origin attacks. CSRF protection for local services must include custom non-standard headers (to trigger CORS preflight) and server-side Origin header validation to ensure the request comes from a trusted local source (e.g., localhost or the app's local origin).
**Prevention:** Always require a custom header (e.g., `X-Matrix-Internal: true`) for sensitive local-only endpoints and validate the `Origin` header against an allowlist (localhost/127.0.0.1). Combine this with masking sensitive tokens in structured logs.
