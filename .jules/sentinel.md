## 2025-05-15 - Localhost CSRF / DNS Rebinding
**Vulnerability:** Malicious websites can use a user's browser to make requests to services on `localhost` (CSRF) or use DNS rebinding to bypass the same-origin policy, potentially leaking sensitive data like authentication tokens.
**Learning:** Standard loopback IP checks (`127.0.0.1`) do not protect against cross-origin browser requests. Requiring a custom non-standard header (e.g., `X-Matrix-Internal`) forces a CORS preflight. However, if the server's CORS policy is too permissive (e.g., allows all origins and all headers), the preflight will succeed and the protection is bypassed. Additionally, failing to detect a remote IP (e.g., `remoteAddress` is undefined) should lead to a secure rejection rather than trusting the header alone.
**Prevention:** 1. Require a custom header for sensitive loopback endpoints. 2. Implement a specialized CORS middleware for these endpoints that only allows trusted local origins (e.g., `http://localhost:*`). 3. Always fail securely if the requester's IP cannot be determined.
